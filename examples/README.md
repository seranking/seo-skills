## How to read an example

Each folder follows the convention `{skill-slug}-{target-slug}-{YYYYMMDD}/`. Inside:

- The **primary deliverable** (`REPORT.md`, `BRIEF.md`, `PLAN.md`, `VERDICT.md`, …) is the synthesised artifact a human reviewer or stakeholder consumes directly. It's verdict-first — the actionable decision is visible in the first 30 lines.
- **Load-bearing files** at the top level (e.g., `keywords.csv`, `issues.csv`, `gaps.csv`, `disavow-candidates.csv`) are the machine-readable artefacts engineering pastes into Jira / spreadsheets.
- **`evidence/`** holds the per-step raw outputs of the skill's `Process` section — preserved for auditability so you can trace any number in the primary back to its source MCP call.

Every primary file lists the SE Ranking + Firecrawl tools used (with exact inputs) in a "Methodology" section, so a reader can re-run any example end-to-end.

## A note on `seo-google`

`seo-google` is the one outlier. The skill is a CLI surface over multiple Google APIs (GSC / PSI / CrUX / Indexing / GA4 / YouTube / NLP) with no single canonical artefact — what gets produced depends entirely on which sub-command you invoke. Its example folder is a **quickstart bundle** showing the shape of a GSC search-analytics report, a PageSpeed Insights report, and a CrUX report, plus a README explaining the credential-tier setup. Treat it as "what each of the 4 credential tiers unlocks" rather than as one finished deliverable.

## Snapshot disclaimer

These outputs are **single-point-in-time snapshots** (dates in folder names). SE Ranking's AI Search prompt database refreshes monthly; share-of-voice numbers, prompt samples, and competitor positioning all drift. Backlink counts, organic-keyword counts, and ranking positions also move continuously. Treat the examples as illustrative of the *shape* of the deliverable, not as a statement about the brands today.
