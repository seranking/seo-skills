## Methodology notes

- **SE Ranking MCP not authenticated in this run.** The skill's primary expansion endpoints (`DATA_getRelatedKeywords`, `DATA_getSimilarKeywords`, `DATA_getLongTailKeywords`, `DATA_getKeywordQuestions`) and the SERP overlap pass (`DATA_getSerpResults`) were unavailable. Volume + KD numbers above are positional estimates from public SERP shape (e.g., Sanity / Strapi / Contentful all targeting these queries with extensive pillar pages = high-volume confirmed; KD inferred from publisher-tier diversity in top-10).
- **SERP overlap was computed from `mcp__firecrawl-mcp__firecrawl_search`** on each of the 8 seed keywords, capturing top-10 organic URLs. Pairwise URL-overlap was computed manually (see `03-cluster-assignment.md`). The 4-6 shared URLs threshold for "same cluster" yielded 0 cross-cluster pairs — every seed forms its own cluster. This is a textbook clean cluster boundary, unusual in practice; reflects that the headless CMS topic is mature with distinct, well-defined sub-topics (definition, comparison, use case, vendor).
- **Estimated 8 SERP-search credits used** (vs ~120-400 SE Ranking SERP-standard credits for an equivalent fetch). Firecrawl search is a cheaper substitute for SERP overlap when SE Ranking is not available.
- **Re-run with SE Ranking authenticated** to get: (a) verified volume and KD numbers, (b) AIO presence per cluster, (c) the actual SERP-task overlap matrix at scale (8 seeds × ~10-15 expansions each = ~80-120 candidate keywords analysed).
