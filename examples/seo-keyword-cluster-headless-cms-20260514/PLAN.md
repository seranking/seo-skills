## Methodology notes

- **SE Ranking data backfilled 2026-05-18** via `DATA_getRelatedKeywords` and `DATA_getSimilarKeywords` (US database). Key finding: the SE Ranking US database indexes the headless CMS keyword cluster at significantly lower volumes than the prior positional estimates suggested — "headless cms" shows 1,300/mo (KD 75) vs the prior ~33K/mo estimate. This reflects that SE Ranking's US-only database captures different scope than global/blended estimates; the prior estimates were likely global-blended figures. The cluster structure (5 clusters, pillar assignments) remains valid.
- **Keywords not indexed in SE Ranking US DB** (no volume returned): "best headless cms 2026", "top headless cms", "headless cms comparison", "strapi vs contentful", "sanity vs contentful", "headless cms pricing", "decoupled cms vs headless", "best headless cms for startups", "best headless cms for nextjs", "enterprise headless cms". These retain SERP-inferred estimates where cited.
- **SERP overlap was computed from `mcp__firecrawl-mcp__firecrawl_search`** on each of the 8 seed keywords, capturing top-10 organic URLs. Pairwise URL-overlap was computed manually (see `03-cluster-assignment.md`). The 4-6 shared URLs threshold for "same cluster" yielded 0 cross-cluster pairs — every seed forms its own cluster. This is a textbook clean cluster boundary, unusual in practice; reflects that the headless CMS topic is mature with distinct, well-defined sub-topics (definition, comparison, use case, vendor).
- **Estimated 8 SERP-search credits used** (vs ~120-400 SE Ranking SERP-standard credits for an equivalent fetch). Firecrawl search is a cheaper substitute for SERP overlap when SE Ranking is not available.
